Frontend — State Management

Cart Context

The CartContext manages cart state globally. Any page can add, remove, or clear cart items through API calls.

// CartContext.jsx
const addToCart = async (product, qty = 1) => {
  if (!user) { alert('Please login'); return; }

  const { data } = await API.post('/cart/add', {
    productId: product._id,
    name:      product.name,
    qty,
    price:     product.price,
    image:     product.images?.[0] || '',
  });
  setCart(data);   // update global state

09
Backend — Security

JWT Authentication

We protect routes using JSON Web Tokens. The protect middleware verifies the token on every request. The admin middleware checks user role.

// authMiddleware.js
export const protect = async (req, res, next) => {
  let token = req.headers.authorization?.split(' ')[1];
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.user = await User.findById(decoded.id).select('-password');
  next();
};

export const admin = (req, res, next) => {
  if (req.user?.role === 'admin') next();
  else res.status(401).json({ message: 'Not authorized as admin' });
};

Backend — Controllers

Order Controller

Controllers handle the business logic. Here the orderController creates orders and links them to the authenticated user.

// orderController.js
export const addOrderItems = async (req, res) => {
  const { orderItems, shippingAddress, totalPrice } = req.body;

  const order = new Order({
    orderItems,
    user:          req.user._id,   // from JWT middleware
    shippingAddress,
    paymentResult: req.body.paymentResult,
    totalPrice,
    isPaid:        true,
  });

  const createdOrder = await order.save();
  res.status(201).json(createdOrder);
};

Database

Why MongoDB?

📄 Flexible Schema

Furniture products have different fields (colors, materials, sizes). MongoDB stores them as flexible JSON documents — no rigid table structure needed.

⚡ Fast Development

With Mongoose, we define schemas in JavaScript. No SQL queries — just JavaScript objects. Perfect for a JS full-stack app.

☁️ Atlas Cloud

MongoDB Atlas provides a free cloud-hosted database. No server setup needed — our backend connects with one connection string in .env.

12
Database — Models

Mongoose Schemas

We have 6 MongoDB collections, each defined as a Mongoose schema:

👤 User

name, email, password (hashed), role (user/admin)

🪑 Product

name, price, description, category, images[], color[], material[], stock

🛒 Cart

user (ref), items[ productId, name, qty, price, image ]

📦 Order

user, orderItems[], shippingAddress, totalPrice, paymentResult, isPaid, status

🎨 CustomOrder

name, email, phone, room, style, budget, dimensions, description

✉️ ContactMessage

name, email, subject, message, timestamps

Database — Example Schema

Custom Order Schema

A great example of MongoDB flexibility — this schema stores a customer's custom furniture consultation request.

// models/CustomOrder.js
const customOrderSchema = new mongoose.Schema({
  name:        { type: String, required: true },
  email:       { type: String, required: true },
  phone:       { type: String },
  room:        { type: String, required: true },  // e.g. Living Room
  style:       { type: String, required: true },  // e.g. Modern
  budget:      { type: String, required: true },  // e.g. $1000 - $3000
  dimensions:  { type: String },
  date:        { type: String },
  description: { type: String, required: true },
}, { timestamps: true });  // adds createdAt & updatedAt automatically

How It All Works Together

Full Data Flow

Example: A user adds a product to cart and places an order.

1️⃣
Login → POST /api/auth/login → Server verifies password with bcryptjs, returns JWT token
2️⃣
Add to Cart → POST /api/cart/add with JWT header → protect middleware verifies → cart saved in MongoDB
3️⃣
Checkout → POST /api/orders → Order saved in MongoDB with payment info and user reference
4️⃣
Admin View → GET /api/orders with admin middleware → All orders retrieved and shown in dashboard
